GERP | April 2022

Type of Project: C++ Program

Inspired after the "grep" command in Linux


SUMMARY

Recently, I teamed up with a partner to create a search engine program.

We architected and implemented the program to allow the user to search

for specific words within files of a chosen directory.

The program then prints out all instances of the word and

where they appear within the directory in this format:

[File path]: [Line #]: Line of txt file where word appears


Project Gerp was engineered to efficiently search across a plethora of

files and is able to index over millions of strings!


The user can either insensitive search for a word ("hello" == "HELLO")

or sensitive search for one.


Insensitive searching for the word "WEASLEY" and sensitive searching for the word "Harry" in every every single book in the Harry Potter series.



PERFORMANCE

When running the program on a directory that contained the Harry

Potter Series, it was able to store all 1,084,170 words and their

location informations into a hashtable in only 0.85 seconds!



ARCHITECTURAL OVERVIEW

Data Structures: Lists & hashtables

Class: Hashtable class

How is the program so efficient/fast? This design makes sure the program would allocate the least memory as possible when storing each word and their location info.

For example, instead of storing the line info as a string in each of the millions of buckets, the streampos of where the line begins in the file is stored, which takes up significantly less space than a long string of words. When the program prints out the line, the file is read until it reaches the streampos and outputs the line.

Additionally, instead of repeating the same filepaths in the hastable, each WordInfo struct stores the filepath as an integer field which points to an index in a vector of strings that contain the corresponding filepath. (Multiple buckets share information).

  • Buckets in the hashtable contain a ProbedNode struct, which contains:

    • key: The lowercase version of each word (simplifies insensitive searches)

    • value: A vector of the instances of that word

  • The instances vector is filled with WordInfo structs that contain:

    • A streampos of the word in the file

    • An index value for the vector of paths that contain filepaths

    • The real word (non - lowercased word)

  • The program reads every single word in all files/subdirectories and hashes it along with its location info into the hashtable.

  • Any time a query is entered, the word is hashed and instances of the word are printed to an output stream provided in the command line.

  • Linear probing was used to hash the strings.
